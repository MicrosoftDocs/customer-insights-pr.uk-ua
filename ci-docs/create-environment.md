# <a name="how-to-create-a-new-environment"></a>Як: Створити нове середовище

Після [придбання ліцензії на Dynamics 365 Customer Insights](paid-license.md) підписку глобальний адміністратор Microsoft 365 клієнта отримує електронний лист із запрошенням створити середовище. Щоб почати, перейдіть до [https://home.ci.ai.dynamics.com/start](https://home.ci.ai.dynamics.com/start). У цьому випадку ви можете перейти безпосередньо до [кроку 1: Надайте основну інформацію](#step-1-provide-basic-information).

Після створення першого середовища глобальний адміністратор Microsoft 365 клієнта може [додавати користувачів зі своєї організації як адміністраторів](permissions.md). Рухаючись вперед, ці адміністратори можуть керувати користувачами та середовищами. Якщо ваша організація купує більше однієї ліцензії для Customer Insights, зв'яжіться з нашою службою [підтримки,](https://go.microsoft.com/fwlink/?linkid=2079641) щоб збільшити кількість доступних середовищ. Щоб отримати додаткові відомості про ємність і ємність додаткових компонентів, перегляньте [посібник із](https://go.microsoft.com/fwlink/?LinkId=866544) ліцензування Dynamics 365.

> [!TIP]
> Якщо ви хотіли б ознайомитися із послугою, див. [Налаштування ознайомлювального середовища](trial-signup.md).

## <a name="prerequisites"></a>вимоги

Для створення середовищ або керування ними в Customer Insights потрібні [дозволи](permissions.md) адміністратора.

## <a name="start-the-environment-creation-process"></a>Запустіть процес створення середовища

1. Відкрийте засіб вибору середовища і виберіть + **Створити**.
  
   :::image type="content" source="media/environment-picker.png" alt-text="Виберіть засіб вибору середовища.":::

1. Дотримуйтеся вказівного досвіду, описаного в наступних розділах, щоб надати всю необхідну інформацію для нового середовища. Якщо ви налаштували середовище раніше, ви також [можете скопіювати конфігурацію](#copy-the-environment-configuration).

## <a name="step-1-provide-basic-information"></a>Крок 1: Надайте основну інформацію

На кроці **Основні відомості** виберіть, чи бажаєте ви створити середовище з нуля, або слід [скопіювати дані з іншого середовища](#copy-the-environment-configuration).

   :::image type="content" source="media/environment-settings-dialog.png" alt-text="Діалог для створення нового середовища Customer Insights.":::

Вкажіть нижчезазначені відомості.

- **Ім'я**: ім'я цього середовища. Це поле буде вже заповнено, якщо ви копіювали з наявного середовища, але ці значення можна змінити.
- **Виберіть рід діяльності**: виберіть основну аудиторію для нового середовища. Ви можете працювати з окремими споживачами (B-to-C) або [бізнес-партнерами](work-with-business-accounts.md) (B-to-B). Якщо ваша організація в основному веде бізнес з фізичними особами, такими як роздрібний продавець або кав'ярня, вибирайте окремих споживачів. Якщо вашою основною аудиторією є інші компанії, наприклад, виробник автомобілів або паперова компанія, вибирайте бізнес-акаунти.
- **Тип**: виберіть тип середовища, яке потрібно створити: виробниче або ізольоване. У ізольованому середовищі не можна планувати заплановане оновлення даних і їх призначено для попередньої реалізації та тестування. У ізольованих середовищах використовується така сама основна аудиторія, як і для вибраного робочого середовища.
- **Регіон**: розташування, в якому розгорнуто та розміщено службу. Щоб [використовувати власний Azure Data Lake Storage обліковий запис](own-data-lake-storage.md) або [підключитися до наявної Microsoft Dataverse організації](customer-insights-dataverse.md), середовище Customer Insights має бути в тому самому регіоні.

## <a name="step-2-configure-data-storage"></a>Крок 2. Налаштування сховища даних

На кроці **зберігання** даних виберіть, де зберігати дані Customer Insights.

Є два варіанти, з яких ви можете вибрати:

- **Зберігання Customer Insights**: Зберіганням даних керує команда Customer Insights. Це опція за замовчуванням, і якщо немає певних вимог для зберігання даних у вашому власному обліковому записі сховища, ми рекомендуємо використовувати цю опцію.
- **Azure Data Lake Storage**: Вкажіть власний Azure Data Lake Storage обліковий запис для зберігання даних, щоб мати повний контроль над тим, де зберігаються дані. Для отримання додаткової інформації дивіться [Використання власного Azure Data Lake Storage облікового запису](own-data-lake-storage.md).

:::image type="content" source="media/data-storage-environment.png" alt-text="Виберіть бажаний варіант зберігання даних.":::

## <a name="step-3-connect-to-microsoft-dataverse"></a>Крок 3. Підключення до Microsoft Dataverse

На кроці **Microsoft Dataverse** ви можете підключити Customer Insights до свого середовища Dataverse. Надавайте спільний доступ до даних, Dataverse щоб використовувати їх із бізнес-програмами на основі, наприклад Dynamics 365 Marketing або програмами, Dataverse керованими моделями, у Power Apps.


Залиште це поле порожнім, якщо у вас немає власного Dataverse середовища, і ми створимо його для вас.

Для отримання додаткової інформації перегляньте [статтю Робота з даними Customer Insights у Microsoft Dataverse](customer-insights-dataverse.md).

:::image type="content" source="media/dataverse-provisioning.png" alt-text="обмін даними з Microsoft Dataverse автоматичним включенням для нових мережевих середовищ.":::

### <a name="step-4-finalize-the-settings"></a>Крок 4. Завершення налаштування

На кроці **Review** пройдіть всі зазначені настройки. Якщо все виглядає нормально, виберіть **Створити**, щоб настроїти середовище.

Ви можете змінити деякі налаштування пізніше. Щоб отримати додаткові відомості, див. розділ [Керування середовищами](manage-environments.md).

## <a name="work-with-your-new-environment"></a>Робота з новим середовищем

Перегляньте наведені нижче статті, які допоможуть розпочати роботу з налаштуванням Customer Insights.

- [Додайте інших користувачів і призначте дозволи](permissions.md).
- [Обробіть джерела даних](data-sources.md) і запустіть ї через [процес уніфікації даних](data-unification.md), щоб отримати [уніфіковані профілі клієнтів](customer-profiles.md).
- [Збагатіть уніфіковані профілі клієнтів](enrichment-hub.md) або [запустіть прогнозні моделі](predictions-overview.md).
- [Створіть сегменти](segments.md) для групування клієнтів і [показників](measures.md) для перегляду KPI.
- [Настройте підключення](connections.md) та [експорти](export-destinations.md) для обробки підмножин даних у інших програмах.

## <a name="copy-the-environment-configuration"></a>Копіювання конфігурації середовища

Як адміністратор, ви можете скопіювати конфігурацію з наявного середовища під час створення нового.

:::image type="content" source="media/environment-settings-dialog.png" alt-text="Знімок екрана: варіанти параметрів у настройках середовища.":::

Відобразиться список усіх наявних середовищ в організації, з яких можна скопіювати дані.

Буде скопійовано параметри конфігурації, перелічені нижче.

- Джерела даних, імпортовані через Power Query
- Конфігурація уніфікації даних
- Сегменти
- Вимірювання
- Зв'язки
- Дії
- Покажчик "Пошук і фільтрація"
- Операції експорту
- Оновити розклад
- Збагачення
- прогноз моделі
- Призначення ролей

## <a name="set-up-a-copied-environment"></a>Настроювання скопійованого середовища

Коли ви копіюєте конфігурацію середовища, вам доведеться виконати кілька додаткових кроків для підтвердження облікових даних:

- Профілі клієнтів. По-перше, аутентифікуйте та проковтніть джерела даних, а також запустіть уніфікацію даних, щоб відтворити профілі клієнтів.
- Облікові дані джерела даних. Ви повинні надати облікові дані для кожної джерело даних, щоб автентифікувати та оновити джерела даних вручну.
- Джерела даних з папки "Загальна модель даних" і Dataverse. Ви повинні створити ці джерела даних вручну з такою ж назвою, як і у вихідному середовищі.
- Секрети зв'язку, які використовуються для експорту і збагачення. Ви повинні переаутенсифікувати зв'язки, а потім знову активувати збагачення та експорт.

Ви побачите повідомлення про підтвердження, коли скопійоване середовище буде створено. Виберіть **Перейти до джерел даних**, щоб переглянути список джерел даних.

Усі джерела даних відображатимуться зі станом **Відсутні облікові дані**. Відредагуйте джерела даних і укажіть облікові дані, щоб оновити їх.

:::image type="content" source="media/data-sources-copied.png" alt-text="Список джерел даних, які було скопійовано, і вони потребують автентифікації.":::

Після оновлення джерел даних перейдіть до розділу **Дані** > **Уніфікувати**. Тут ви знайдете настройки з вихідного середовища. Відредагуйте їх відповідно до потреб або виберіть **Виконати**, щоб почати процес уніфікації даних і створити єдину сутність клієнта.

Після завершення уніфікації даних виберіть **Показники** та **Сегменти**, щоб оновити їх також.

Перш ніж повторно активувати експорт і збагачення, перейдіть до **розділу Підключення адміністратора** > **·**, щоб повторно ввизначити зв'язки в новому середовищі.
