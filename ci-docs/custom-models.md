# <a name="custom-machine-learning-models"></a>Спеціальні моделі машинного навчання

> [!NOTE]
> Підтримка машинне навчання Studio (класична) завершиться 31 серпня 2024 року. Ми рекомендуємо вам перейти на Azure машинне навчання [до](/azure/machine-learning/overview-what-is-azure-machine-learning) цієї дати.
>
> З 1 грудня 2021 року ви не зможете створювати нові машинне навчання студійних (класичних) ресурсах. До 31 серпня 2024 року ви можете продовжувати користуватися наявними ресурсами машинне навчання Studio (класичний). Для отримання додаткових відомостей дивіться [Міграція до Azure машинне навчання](/azure/machine-learning/migrate-overview).

Користувацькі моделі дозволяють керувати робочими процесами на основі моделей Azure машинне навчання. Робочі цикли допомагають вибрати дані, для яких потрібно створити аналітичні огляди та зіставити результати з уніфікованими даними про клієнта. Для отримання додаткових відомостей про створення настроюваних моделей ML див. розділ [Використання моделей на основі машинного навчання Azure](azure-machine-learning-experiments.md).

## <a name="prerequisites"></a>вимоги

- Веб-служби, опубліковані через [Azure машинне навчання конвеєри](/azure/machine-learning/concept-ml-pipelines).
- Трубопровід повинен бути опублікований під кінцевою точкою [трубопроводу](/azure/machine-learning/how-to-run-batch-predictions-designer#submit-a-pipeline-run).
- Обліковий [запис сховища Azure Data Lake Gen2, пов’язаний](/azure/storage/blobs/data-lake-storage-quickstart-create-account) із екземпляром Azure Studio.
- Для Azure машинне навчання робочі області з конвеєрами, дозволами власника або адміністратора доступу користувачів до робочої області Azure машинне навчання.

  > [!NOTE]
  > Дані передаються між екземплярами Customer Insights і вибраними веб-службами або воронками продажів Azure в робочому циклі. Під час передавання даних до служби Azure переконайтеся, що служба налаштована на обробку даних з урахуванням способу й розташування, необхідних для забезпечення відповідності юридичним або нормативним вимогам для цих даних для організації.

## <a name="add-a-new-workflow"></a>Додати новий робочий цикл

1. Перейдіть до розділу **Аналітика** > **Спеціальні моделі** та виберіть елемент **Новий робочий цикл**.

1. Надайте впізнаване **ім’я**.

   :::image type="content" source="media/new-workflowv2.png" alt-text="Знімок екрана області «Новий робочий цикл».":::

1. Виберіть організацію, яка містить веб-службу в розділі **Клієнт, що містить веб-службу**.

1. Якщо передплата на Azure Machine Learning знаходиться в іншому клієнті, ніж Customer Insights, виберіть **Вхід** зі своїми обліковими даними для вибраної організації.

1. Виберіть **Робочі області**, зв'язані з веб-службою.

1. Виберіть конвеєр Azure машинне навчання у веб-службі **, яка містить розкривний список моделі**. Потім виберіть **Далі**.
   Докладніше про [публікацію воронки продажів у машинному навчанні Azure за допомогою конструктора](/azure/machine-learning/concept-ml-pipelines#building-pipelines-with-the-designer) або [SDK](/azure/machine-learning/concept-ml-pipelines#building-pipelines-with-the-python-sdk).

1. Для кожного **Введення Веб-служби** виберіть відповідну **Сутність** із Customer Insights. Потім виберіть **Далі**.
   > [!NOTE]
   > Робочий цикл настроюваної моделі застосовуватиме евристику для зіставлення полів вводу веб-служби із атрибутами сутності на основі імені та типу даних поля. З'явиться повідомлення про помилку, якщо поле веб-служби не вдасться зіставити із сутністю.

   :::image type="content" source="media/intelligence-screen2-updated.png" alt-text="Налаштуйте робочий цикл.":::

1. Для **параметрів** виводу моделі встановіть такі властивості:
   - **Назва** сутності для результатів виводу трубопроводу
   - **Введіть ім’я** параметра сховище даних вашого конвеєра
   - **Ім’я** параметра "Шлях виводу" пакетного конвеєра

   :::image type="content" source="media/intelligence-screen3-outputparameters.png" alt-text="Область параметрів виводу моделі.":::

1. Виберіть атрибут відповідності з **ідентифікатора клієнта в результатах,** які ідентифікують клієнтів, і натисніть кнопку **Зберегти**.

   :::image type="content" source="media/intelligence-screen4-relatetocustomer.png" alt-text="Пов’язати результати з областю даних клієнта.":::

   На **екрані "Збережено** робочий цикл" відображаються відомості про робочий цикл. Якщо для конвеєра Azure машинне навчання настроєно робочий цикл, Customer Insights приєднається до робочої області, яка містить конвеєр. Customer Insights отримає роль співавтора **в** робочій області Azure.

1. Виберіть **Готово**. Відобразиться **сторінка Користувацькі моделі**.

1. Виберіть вертикальні три крапки (&vellip;) для робочого циклу та натисніть кнопку **Виконати**. Робочий цикл також запускається автоматично з кожним [запланованим оновленням](schedule-refresh.md).

## <a name="manage-an-existing-workflow"></a>Керування наявним робочим циклом

Перейдіть до розділу **Інтелектуальні** > **настроювані моделі**, щоб переглянути створені робочі цикли.

Виберіть робочий цикл, щоб переглянути доступні дії.

- **Редагування** робочого циклу
- **Запуск** робочого циклу
- [**Видалення**](#delete-a-workflow) робочого циклу

### <a name="edit-a-workflow"></a>Редагуйте робочий цикл

1. Перейдіть до **користувальницьких моделей розвідки** > **·**.

1. Поруч із робочим циклом, який потрібно оновити, виберіть вертикальні три крапки (&vellip;) і натисніть кнопку **Редагувати**.

1. За потреби змініть **коротке ім’я** та натисніть кнопку **Далі**.

1. Для кожного **введеного веб-служби** оновіть відповідну **сутність** з Customer Insights, якщо потрібно. Потім виберіть **Далі**.

1. Для **параметрів** виводу моделі змініть одне з наступного:
   - **Назва** сутності для результатів виводу трубопроводу
   - **Введіть ім’я** параметра сховище даних вашого конвеєра
   - **Ім’я** параметра "Шлях виводу" пакетного конвеєра

1. Змініть атрибут відповідності з **ідентифікатора клієнта в результатах пошуку**, щоб ідентифікувати клієнтів. Виберіть атрибут у виводі зі значеннями, подібними до стовпця «Ідентифікатор клієнта» сутності клієнта. Якщо такого стовпця немає в наборі даних, виберіть атрибут, який однозначно ідентифікує рядок.

1. Виберіть **Зберегти**

### <a name="delete-a-workflow"></a>Видаліть робочий цикл

1. Перейдіть до **користувальницьких моделей розвідки** > **·**.

1. Поруч із робочим циклом, який потрібно видалити, виберіть вертикальні три крапки (&vellip;) і натисніть кнопку **Видалити**.

1. Підтвердьте видалення.

Робочий цикл буде видалено. [Сутність](entities.md), створена під час створення робочого циклу, зберігається, і її можна переглянути на **сторінці "Сутності даних** > **"**.

## <a name="view-the-results"></a>Переглянути результати

Результати робочого процесу зберігаються в імені сутності, визначеному для **параметрів** виводу моделі. Отримайте доступ до цих даних зі сторінки [**·** > **«Суб’єкти даних**»](entities.md) або за допомогою [доступу до](apis.md) API.

### <a name="api-access"></a>Доступ до API

Для певного запиту OData для отримання даних із сутності настроюваної моделі скористайтеся таким форматом:

`https://api.ci.ai.dynamics.com/v1/instances/<your instance id>/data/<custom model output entity name>%3Ffilter%3DCustomerId%20eq%20'<guid value>'`

1. Замініть `<your instance id>` ідентифікатором середовища Customer Insights, який відображається в адресному рядку браузера під час доступу до Customer Insights.

1. Замініть `<custom model output entity>` на ім’я сутності, яке ви надали для параметрів **виводу** моделі.

1. Замініть `<guid value>` його на ідентифікатор клієнта, до якого потрібно отримати доступ. Цей ідентифікатор відображається на [сторінці](customer-profiles.md) профілів клієнтів у полі "Ідентифікатор клієнта".

## <a name="frequently-asked-questions"></a>Запитання й відповіді

- Чому не вдається переглянути воронку продажів під час настроювання робочого циклу настроюваної моделі?
  Ця проблема часто виникає через проблему конфігурації воронки продажів. Переконайтеся, що [настроєно вхідні параметри](azure-machine-learning-experiments.md#dataset-configuration), і також налаштовано [вихідні параметри сховища даних і переліку дій](azure-machine-learning-experiments.md#import-pipeline-data-into-customer-insights).

- Що означає повідомлення про помилку «Не вдалося зберегти робочий цикл аналітики»? 
  Користувачі зазвичай бачать це повідомлення про помилку, якщо в робочій області немає прав «Відповідальний» або «Адміністратор доступу користувача». Користувачу потрібний вищий рівень дозволів, щоб дозволити Customer Insights обробляти робочий цикл як службу, а не використовувати облікові дані користувача для подальших виконань робочого циклу.

- Чи підтримується приватна кінцева точка / приватне посилання для робочого процесу моєї користувацької моделі?
  Customer Insights наразі не підтримує приватну кінцеву точку для користувацьких моделей з коробки, але доступний спосіб вирішення проблеми вручну. Будь ласка, зв’яжіться зі службою підтримки для отримання детальної інформації.

## <a name="responsible-ai"></a>Відповідальний ШІ

Прогнози надають можливості для створення кращого досвіду клієнтів, поліпшення бізнес-ресурсів та потоків доходів. Ми настійно рекомендуємо збалансувати цінність вашого прогнозу з його впливом та упередженнями, які можуть бути введені в етичному порядку. Дізнайтеся більше про те, як корпорація Microsoft [звертається до відповідального ШІ](https://www.microsoft.com/ai/responsible-ai?activetab=pivot1%3aprimaryr6). Також можна дізнатися про [техніки та процеси для відповідального машинного навчання](/azure/machine-learning/concept-responsible-ml), що стосуються машинного навчання Azure.
