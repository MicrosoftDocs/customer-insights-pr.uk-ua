## <a name="may-2021-updates"></a>Оновлення за травень 2021

Оновлення в травні 2021 року містять кілька функцій, оновлення продуктивності та виправлення помилок.

### <a name="data-ingestion"></a>Приймання даних

- **Перегляньте або змініть метадані або визначення сутності під час вкладення даних з Azure Data Lake Storage** Тепер можна переглядати та редагувати метадані або визначення сутності в аналізі аудиторії під час вкладення даних з папки Common Data Model до Azure Data Lake Storage. Ця можливість дозволяє отримувати відгуки в режимі реального часу, перевіряти модель та знаходити помилки. Ви можете без перешкод редагувати як model.json, так і manifest.json.

### <a name="extensibility"></a>Розширення

- **Вдосконалений експорт сегментів, користувацькі розклади та дублювання**: тепер ви можете [побачити усі експорти для певного сегмента](export-destinations.md#view-exports-and-export-details) у вигляду списку. Це нове подання допомагає керувати способом використання певного сегмента та скоригувати наявні або створити нові експорти.    
  Ви можете [визначити користувацький розклад оновлень](export-destinations.md#schedule-and-run-exports) для окремого експорту або ж одразу для кількох експортів. Досі всі експорти запускалися під час кожного оновлення системи.    
  Замість створення нового експорту з нуля ви можете почати, взявши за основу наявний експорт, і зберегти трохи часу.

- **Експорт сегментів до Microsoft Advertising** Відтепер призначення експорту включають Microsoft Advertising. Створюйте аудиторії Customer Match у Microsoft Advertising на основі уніфікованих даних із профілів клієнтів і використовуйте ці аудиторії для своїх рекламних кампаній. Для отримання додаткових відомостей див. [Експорт сегментів до Microsoft Advertising](export-microsoft-advertising.md).

- **Експорт сегментів до LinkedIn Ads** Відтепер призначення експорту включають LinkedIn Ads, отже ви можете розблокувати націлювання на контактних осіб та націлювання на компанії за допомогою LinkedIn, експортуючи дані уніфікованих профілів ваших клієнтів. Для отримання додаткових відомостей див. [Експорт сегментів до LinkedIn Ads](export-linkedin-ads.md).


- **Експорт сегментів до Omnisend** Відтепер призначення експорту включають Omnisend. Застосовуйте створені в аналізі аудиторії сегменти, щоб формувати кампанії, здійснювати маркетингові розсилки електронною поштою та використовувати конкретні групи клієнтів за допомогою Omnisend. Для отримання додаткових відомостей див. [Експорт сегментів до Omnisend](export-omnisend.md)

### <a name="predictions"></a>Прогнози

- **Звіт про зручність використання вхідних даних** У звіті про зручність використання вхідних даних надається узагальнене подання помилок і попереджень, які можуть виникати при роботі готових прогнозів. Він також дає рекомендації щодо поліпшення продуктивності моделі.    
  Звіт стає доступним після того, як модель завершить процес навчання. Він створюється окремо для кожної моделі, незалежно від того, чи була робота моделі успішною, чи ні.
  Наразі ця функція доступна лише для моделі відтоку транзакцій. Для отримання додаткових відомостей див. [Звіт про зручність використання даних](manage-predictions.md#input-data-usability-report).

### <a name="relationships"></a>Зв'язки

- **Засіб графічного відображення зв'язків** Подання графічного відображення зв'язків дозволяє побачити всі наявні зв'язки між сутностями та їх кратність. Тепер зв'язки впорядковані за групами: створені користувачем, системні та успадковані зв'язки. Це подання можна також експортувати як зображення. Для отримання додаткових відомостей див. [Перегляд зв’язків](relationships.md#view-relationships). 

## <a name="april-2021-updates"></a>Оновлення за квітень 2021 р.

Оновлення в квітні 2021 року містять кілька функцій, оновлення продуктивності та виправлення помилок.

### <a name="data-unification"></a>Уніфікація даних
 
- **Вдосконалена робота злиття для уніфікації даних**    
  
   Ми вдосконалили процедуру налаштування користувачами злиття в процесі уніфікації даних. Зміни включають зрозуміле впорядкування об’єднаних полів та докладну статистику поєднаних та одинарних полів.

- **Зміна впорядкування сутностей та налаштування усіх записів-джерел до сутності Клієнт**  
      
   Тепер у процесі уніфікації даних можна змінювати порядок і видаляти сутності в наявному плані об’єднання даних. Це дозволяє гнучко змінювати порядок сутностей в процесі зіставлення відповідно до потреб бізнесу. Крім того, ми дозволили вносити усі незіставлені записи до кінцевої сутності *Клієнт*, що дає їм змогу вказувати власне визначення набору даних профілю клієнта.

### <a name="enrichments"></a>Збагачення

 - **Нове збагачення: вдосконалені адреси**    
  
   Ми раді представити нове збагачення для вдосконалення адрес у ваших даних клієнтів. Адреси в даних можуть бути неструктуровані, неповні або неправильні. Ця функція використовує моделі Microsoft, щоб нормалізувати та збагачувати адреси, приводячи їх до формату Common Data Model для забезпечення вищої точності та якості аналітичних висновків.
 
   Для отримання додаткових відомостей див. [Збагачення профілів клієнтів вдосконаленими адресами](enrichment-enhanced-addresses.md).

- **Інтерактивне налаштування для збагачень**    
  
   Ми переглянули процедуру налаштування для збагачень, додавши простий інтерактивний інтерфейс. Тепер використовується чіткий покроковий процес створення та редагування збагачень.
 
   Крім того, ми відокремили налаштування підключень для сторонніх збагачень, щоб одне підключення можна було використовувати різними збагаченнями. Тільки адміністратори можуть налаштовувати нові підключення, але створені підключення доступні як для адміністраторів, так і для співавторів.    

   Щоб отримати докладніші відомості, див. [Огляд підключень](connections.md).

- **Кілька збагачень одного й того самого типу**    
  
   Тепер ми дозволяємо користувачам створювати кілька збагачень однакового типу збагачення та керувати ними. Наприклад, можна створити два окремі збагачення адрес, щоб збагатити два різні сегменти клієнтів. Для різних типів збагачень існують різні обмеження щодо кількості збагачень одного типу, які можна створити.
  
   Для отримання додаткових відомостей див. розділ [Збагачення профілів клієнтів](enrichment-hub.md).

## <a name="march-2021-updates"></a>Оновлення за березень 2021 року

Оновлення в березні 2021 року містять кілька функцій, оновлення продуктивності та виправлення помилок.

### <a name="activities"></a>Справи

- **Майстер справ і семантичні типи**

   Ми вдосконалили та оновили процедуру зіставлення справ, щоб створювати зіставлення справ було простіше. Тепер користувачі бачитимуть інтерфейс із підказками, які допоможуть виконати кожен з кроків процесу. На кроці зіставлення справ, на додаток до вибору з багатьох типів справ користувач тепер може обрати шлях семантичного зіставлення даних для *Передплати* та/або *SalesOrderLine* зі стандартними для галузі схемами, які можуть використовуватися для споживання на наступних ланках бізнес-процесу.   

   Додаткові відомості див. в розділі [Справи клієнта](activities.md).

### <a name="data-ingestion"></a>Приймання даних

- **Підключення до локальних джерел даних за допомогою потоків даних і шлюзів Power Platform** Ми раді повідомити про підготовчу версію потоків даних Power Platform та локального підключення за допомогою шлюзів у Customer Insights із пов'язаним середовищем Power Platform або Dataverse. Усі нові джерела даних, створені в середовищі Customer Insights зі зв'язаним середовищем Dataverse, за замовчуванням використовуватимуть потоки даних Power Platform, завдяки чому з'явиться можливість підключатися до локальних даних та використовувати широкий ряд з'єднувачів та можливостей перетворення.

### <a name="extensibility"></a>Розширення

- **Експорти поділено на підключення та експорти** Ми змінили назву сторінки **Призначення експорту** на **Підключення** й додали окрему сторінку для **Експортів**. У межах цього оновлення ми перетворюємо існуючі експорти на пари з підключення та експорту, що використовує це підключення. Тепер адміністратори мають більш зрозуміле відображення вихідних даних на сторінці **Підключення**. Усі ролі користувачів мають доступ до сторінки **Експорти**, але лише адміністратори можуть дозволити співавторам редагувати певні експорти зі спільними підключеннями.     
  Додаткові відомості: [Огляд компонентів](connections.md) і [Огляд експортів](export-destinations.md).

- **Експорт сегментів до Campaign Monitor** Відтепер призначення експорту включають Campaign Monitor. Тепер ви можете експортувати сегменти з Customer Insights до списків Campaign Monitor та використовувати їх як основу для своїх маркетингових кампаній.    
   Додаткові відомості: [Експортування даних в Campaign Monitor](export-campaign-monitor.md).

- **Експорт сегментів до Constant Contact** Відтепер призначення експорту включають Constant Contact. Тепер ви можете експортувати сегменти з Customer Insights до списків Constant Contact та використовувати їх як основу для своїх маркетингових кампаній.   
   Додаткові відомості: [Експортування даних в Constant Contact](export-constant-contact.md).

- **Експорт сегментів до RollWorks** Відтепер призначення експорту включають RollWorks. Тепер ви можете експортувати сегменти з Customer Insights до аудиторій RollWorks та використовувати їх як основу для своєї реклами у корпоративному сегменті.    
   Докладні відомості див. в статті [Експортування даних в RollWorks](export-rollworks.md).

- **Експорт сегментів до Snapchat** Відтепер призначення експорту включають Snapchat. Тепер ви можете експортувати сегменти з Customer Insights до аудиторій Snapchat та використовувати їх як основу для рекламної активності.     
   Докладні відомості див. в статті [Експортування даних в Snapchat](export-snapchat.md).

### <a name="predictions"></a>Прогнози

- **Використовуйте фільтри у прогнозних рекомендаціях продуктів** Ми додали можливість використовувати фільтри продуктів у нашій моделі рекомендацій продуктів. Тепер можна створити прогноз, який використовуватиме певну підмножину ваших продуктів.    
   Додаткові відомості: [Налаштування фільтрів продуктів](predict-product-recommendation.md#configure-product-filters).

- **Створення сегментів з прогнозів моделі** Ми додали швидкий спосіб створення сегментів з результатів моделі прогнозування. На сторінці результатів моделі можна легко створити новий сегмент, вибравши нову можливість **Створити сегмент**.    
  Додаткові відомості: [Створення сегмента на основі моделі прогнозування](prediction-based-segment.md).

- **Пояснення щодо рекомендацій продуктів** Ми додали відомості, що описують ключові фактори, вивчені моделлю ШІ для створення рекомендацій продуктів, та те, наскільки сильно ці фактори впливають на ймовірність потрапляння продукту до рекомендацій. Ці відомості тепер містяться на сторінці із результатами роботи моделі.    
   Для отримання додаткових відомостей див. розділ [Перегляд стану та результатів прогнозу](predict-product-recommendation.md#review-a-prediction-status-and-results).

## <a name="february-2021-updates"></a>Оновлення за лютий 2021 року

Оновлення у лютому 2021 року містять кілька нових функцій, удосконалення продуктивності та виправлення помилок.

#### <a name="extensibility"></a>Розширення

- **Експортувати сегменти до AdRoll**

  Ми розширили призначення експорту, щоб включити AdRoll. Тепер можна експортувати сегменти з Customer Insights до аудиторій AdRoll і використовувати їх як базову модель для реклами. Для отримання додаткових відомостей див. [З’єднувач для AdRoll](export-adroll.md).

#### <a name="segments"></a>Сегменти
 
- **Дублювати сегмент**
  
  Щоб створити новий сегмент на основі наявного, тепер можна створити і відредагувати сегмент, що повторюється, щоб узгодити його додатково. 

- **Додавання до сегмента додаткових атрибутів**

  Тепер до виводу сегмента можна включати атрибути, навіть якщо ці атрибути не є частиною профілю клієнта. Наприклад, додайте ідентифікатори передплати в сегмент, навіть якщо він входить до сутності передплати, яка має відношення M:1 до сутності клієнта. Якщо атрибут належить до сутності, пов'язаної із сутністю клієнта, можна включити ці атрибути.  

#### <a name="predictions"></a>Прогнози

- **Створення прогнозних рекомендацій продуктів**

  Розуміння того, у придбанні чого зацікавлені клієнти, є одним перших кроків, необхідних для збільшення доходів від бізнесу і підвищення лояльності клієнтів завдяки персоналізації та взаємодії. Надання рекомендацій щодо продуктів, які не відповідають інтересам вашого клієнта, можуть створити відчуття роз'єднання між клієнтом і вашим бізнесом, а також обмежити загальний потенційний дохід і досвід для клієнта. 

  Використовуючи власні дані, тепер можна створювати прогнози щодо продуктів, які клієнти можуть придбати в майбутньому. Для отримання додаткових відомостей див. [Прогноз рекомендації продуктів](predict-product-recommendation.md).

#### <a name="system-administration"></a>Адміністрування системи

- **Середовище копіювання підтримує більше типів джерел даних**

  Адміністратори можуть копіювати конфігурації середовища до нового середовища в межах однієї організації. Ця функція розширює функціональні можливості середовища копіювання для інцидентів, в яких використовуються джерела даних на основі озера даних Common Data Service або папки Common Data Model.

## <a name="january-2021-updates"></a>Оновлення за січень 2021 р.

Оновлення у січні 2021 року містять кілька нових функцій, удосконалення продуктивності та виправлення помилок.

#### <a name="extensibility"></a>Розширення

- **Розширена функціональність і підвищена продуктивність для експорту SFTP** Тепер ви можете експортувати на хост SFTP всі вихідні сутності з Customer Insights. Раніше експорт обмежувався сутностями сегмента. Крім того, продуктивність експорту SFTP дозволяє отримати більше даних за менший час, залежно від продуктивності вашого хосту SFTP.    
  Для отримання додаткових відомостей див. [З’єднувач для SFTP (підготовча версія)](export-sftp.md).  

#### <a name="segments"></a>Сегменти

- **Пропозиції сегментів на основі машинного навчання для поліпшення показників** З’явився новий спосіб виявляти та створювати сегменти. Система використовує модель ШІ для надання пропозицій сегментів, які можуть допомогти покращити KPI (показники), котрі ви вже відстежуєте. Ми показуємо ступінь впливу атрибутів, вибраних вами, на показник або інший основний атрибута. Ці відомості допомагають знаходити перспективні сегменти для потенційних угод.    
  Для отримання додаткових відомостей див. [Пропозиції сегментів (підготовча версія)](suggested-segments.md).

#### <a name="data-unification"></a>Уніфікація даних

- **Покращення у роботі із зіставленнями** Оновлено процедуру зіставлення в області уніфікації даних. Тепер ви можете налаштовувати та переглядати правила зіставлення, зокрема докладні характеристики, які краще пояснюють роботу зіставлення. З’явилися можливості відключення правил зіставлення, щоб воно більше не враховувалося при збереження конфігурації, перетягування правил зіставлення тощо.
  Для отримання додаткових відомостей див. розділ [Зіставлення сутностей](match-entities.md).

- **Результат вилучення дублікатів у процесі зіставлення доступний у формі сутності** Результат вилучення дублікатів у процесі зіставлення тепер записується в окрему сутність для подальшого аналізу. Ця сутність складається з полів, які використовуються в процесі вилучення дублікатів, а також запису-переможця і відповідних альтернативних записів, котрі було об’єднано із записом-переможцем.
  Для отримання додаткових відомостей див. [Результати вилучення дублікатів як сутність](match-entities.md#deduplication-output-as-an-entity).

#### <a name="system-administration"></a>Адміністрування системи

- **Легке надсилання даних до Microsoft Dataverse** Тепер ви можете ділитися результатами Customer Insights із програмами Microsoft Dataverse, використовуючи керований Data Lake Microsoft Dataverse. Після того, як ви пов’яжете середовище Dataverse із Customer Insights, ви зможете дозволити обмін даними.
  Щоб отримати додаткові відомості, див. розділ [Керування середовищами](manage-environments.md).
