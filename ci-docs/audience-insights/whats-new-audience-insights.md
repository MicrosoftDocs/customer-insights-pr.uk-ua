## <a name="november-2020-updates"></a>Оновлення за листопад 2020 р.

Оновлення в листопаді 2020 містять кілька нових функцій, оновлення продуктивності і виправлення помилок.

### <a name="new-and-updated-features-in-november-2020"></a>Нові та оновлені функції в листопаді 2020

#### <a name="data-enrichment"></a>Збагачення даних

- **Вводьте власні дані збагачень, використовуючи настроюваний імпорт за протоколом SFTP**
  
  За допомогою настроюваного імпорту SFTP можна імпортувати дані збагачення, які не мають обов’язково проходити процес уніфікації даних. Дізнайтеся більше про настроюваний імпорт SFTP.

  Для отримання додаткових відомостей див. [Збагачення профілів користувачів настроюваними даними (підготовча версія)](enrichment-SFTP-custom-import.md).
 
- **Збагачення даних клієнтів за допомогою даних розташування від HERE Technologies**

  За допомогою служб збагачення даних HERE Technologies можна отримати точніше розуміння відомостей про місцезнаходження клієнтів, використовуючи нормалізацію адрес, отримання довготи та широти тощо. Дізнайтеся більше про збагачення із використанням HERE Technologies.

  Для отримання додаткових відомостей див. [Збагачення профілів користувачів із використанням HERE Technologies](enrichment-here.md).

#### <a name="data-unification"></a>Уніфікація даних

- **Гнучке профілювання даних лише для вибраних сутностей та полів з вашого облікового запису сховища**

  Можна вказати, для яких сутностей та полів даних у папці Common Data Model у вашому обліковому записі сховища Azure Data Lake необхідно увімкнути профілювання даних в межах процесу приймання даних.

  Для отримання додаткових відомостей див. [Підключення до папки Common Data Model](connect-common-data-model.md#connect-to-a-common-data-model-folder).

#### <a name="extensibility"></a>Розширення

- **Активація сегментів за допомогою Google Ads**

  Експортуйте сегменти з до списків аудиторій Google Ads та використовуйте ці списки для рекламування у Google Search, Google Display Network, YouTube та Gmail. Дізнайтеся більше про активацію сегментів за допомогою Google Ads.

  Для отримання додаткових відомостей див. [З’єднувач для Google Ads](export-google-ads.md).

- **Активація сегментів за допомогою Marketo**

  Експортуйте сегменти до аудиторій Marketo і користуйтеся цими аудиторіями для автоматизації маркетингу. Дізнайтеся більше про активацію сегментів за допомогою Marketo. 

  Для отримання додаткових відомостей див. [З’єднувач для Marketo](export-marketo.md).

- **Активація сегментів за допомогою DotDigital**

  Експортуйте сегменти до DotDigital і використовуйте їх для цілей маркетингу. Дізнайтеся більше про активацію сегментів за допомогою DotDigital. 

  Для отримання додаткових відомостей див. [З’єднувач для DotDigital](export-dotdigital.md).

#### <a name="predictions"></a>Прогнози

- **Прогнозування відтоку транзакцій**

  Функція прогнозування відтоку транзакцій дає змогу, не звертаючись по допомогу до спеціаліста з аналізу даних, передбачити вірогідність того, що клієнт припинить придбання продуктів або послуг.  Використовуючи оцінки прогнозу, ви можете поєднувати інші відомості щодо ваших користувачів, наприклад, значення клієнта, щоб створювати сегменти із високим ризиком відтоку або значущими клієнтами. За допомогою цього сегмента ви зможете спрямувати маркетингові заходи, підтримку користувачів та інші сценарії, безпосередньо на потрібних користувачів і, таким чином, зменшити ризик відтоку.
 
  Налаштуйте визначення відтоку як вікно, що залежить від часу, котрий базується на особливостях вашого бізнесу, а також визначте, коли вважатиметься, що відбувся відтік для певного користувача. Наприклад, продуктовий магазин може розглядати клієнта, як такого, котрий відвернувся, якщо той не купував нічого протягом 30 днів.
 
  На пізніших етапах при створенні прогнозу ми покажемо, які дані є необхідними, та дамо можливість зіставляти дані, що стосуються вашого бізнесу, із полями, потрібними для прогнозування відтоку ваших клієнтів. Ви також можете задати розклад, за яким відбуватиметься перенавчання моделі на основі відомостей у вашій системі, щоб модель адаптувалася до мінливих бізнес-умов.
 
  Для отримання додаткових відомостей див. [Прогнозування відтоку транзакцій (підготовча версія)](predict-transactional-churn.md).

#### <a name="system-administration"></a>Адміністрування системи

- **Скинути середовище**

  Скидання всього в середовищі вибраної інсталяції, щоб почати усе заново.

  Для отримання додаткових відомостей див. [Скидання наявного середовища](manage-environments.md#reset-an-existing-environment).


- **Підключення до облікового запису сховища Azure Data Lake із використанням принципала служби**

  Зберігайте отримані результати та читайте дані з вашому облікового запису сховища, використовуючи принципал служби Azure. Наявні підключення до облікового запису сховища можуть продовжувати використовувати ключ облікового запису. Крім того, вони пропонують можливість оновлення, щоб у подальшому використовувати принципал служби. Нові підключення базуватимуться на методі автентифікації на основі принципала служби для облікового запису сховища.

  Для отримання додаткових відомостей див. [Підключення до облікового запису Azure Data Lake Storage Gen2 за допомогою принципала служби Azure для аналізу аудиторій](connect-service-principal.md).

## <a name="october-2020-updates"></a>Оновлення за жовтень 2020 року

Оновлення в жовтні 2020 містять кілька нових функцій, оновлення продуктивності і виправлення помилок.

### <a name="new-and-updated-features-in-october-2020"></a>Нові та оновлені функції в жовтні 2020

#### <a name="extensibility"></a>Розширення

- **Експорт до MailChimp**

Експортуйте сегменти до наявних списків аудиторій в MailChimp, щоб використовувати персоналізовані можливості електронної пошти для своїх клієнтів.

Для отримання додаткових відомостей див. [З’єднувач для MailChimp](export-mailchimp.md).

#### <a name="data-enrichment"></a>Збагачення даних

- **Вилучення дублікатів вихідних записів у сутності зіставлення**

Укажіть правила вилучення дублікатів для сутностей, які використовуються в процесі зіставлення, щоб визначити повторювані записи. Об'єднайте їх у єдиний запис та зв'яжіть всі вихідні записи із цим об’єднаним записом. Цей запис, отриманий після вилучення дублікатів, використовуватиметься надалі у процесі перехресного зіставлення сутностей.

Для отримання додаткових відомостей див. [Визначення правил вилучення дублікатів для сутності зіставлення](match-entities.md#define-deduplication-on-a-match-entity).

#### <a name="system-administration"></a>Адміністрування системи

- **Оркестрація: нова можливість оновлення у Злитті**

До сьогоднішнього дня при виконанні процесу злиття система запускала всі подальші процеси, які залежать від злиття та пов’язаних процесів. Тепер можна перевірити результати роботи процесу злиття (уніфіковану сутність клієнта), перш ніж використовувати їх у подальших операціях, наприклад, для створення сегментів або показників.
На сторінці злиття тепер можна за вибором запустити та виконати процес злиття окремо. Для того, щоб також оновити й усі подальші процеси, ви можете вибрати запуск процесу злиття та усіх подальших процесів. 

## <a name="september-2020-updates"></a>Оновлення за вересень 2020 р.

Оновлення у вересні 2020 року містять кілька функцій, оновлення продуктивності і виправлення помилок.

### <a name="new-and-updated-features-in-september-2020"></a>Нові та оновлені функції у вересні 2020

#### <a name="activities"></a>Дії

- **інтелектуальне прогноз семантики атрибутів**

Ця нова функція передбачає семантичні типи вхідних атрибутів, які передаються до процесу уніфікації даних. Вона використовує моделі машинного навчання, які покращують точність і заощаджують час.

#### <a name="enrichments"></a>Збагачення

- **Демографічні збагачення від Experian**

Демографічні збагачення з Experian тепер доступні в підготовчій версії. Компанія Experian є світовим лідером у галузі звітування та маркетингу щодо споживчих та ділових кредитів. За допомогою [послуг компанії Experian щодо збагачення даних](https://www.experian.com/marketing-services/microsoft?cmpid=ems_web_mci_cdppage) можна глибше розуміти своїх клієнтів, збагачуючи профілі клієнтів демографічними даними, такими як розмір сім'ї, прибуток тощо.

Щоб скористатися цією функцією, слід мати активну передплату Experian.

Для отримання додаткових відомостей див. розділ [Збагачення профілів клієнтів з демографічними даними Experian](enrichment-experian.md)


#### <a name="system-administration"></a>Адміністрування системи

- **Область відомостей про завдання**

В області відомостей про завдання можна переглянути докладні відомості про завдання, які виконує система. Це зручний засіб для визначення проблем з конфігурацією та пошуком рішень.
Перегляньте повідомлень про помилки, щоб дізнатися, як усунути можливі проблеми.
 
- **Обробка інформації, доданої до додаткових сторінок**

Це вдосконалення додає відомості про стан сутностей на сторінці **Сутності** та **Клієнти**.
 
Крім того, можна знайти докладну інформацію про перебіг процесів, а також відомості про завдання на обох цих сторінках.

- **Вдосконалення сторінки стану системи**

Під час перегляду експорту даних було вдосконалено структуру таблиці відомостей про стан в розділі **Система** > **Стан**.
 
Крім того, помилки в стовпці **Докладно** тепер є більш докладними і дієвими. 
 
- **Скасування повертає завдання до попереднього стану**

У разі скасування завдання, наприклад, у процесі зіставлення, його буде знову змінено до останнього стану. Наприклад, якщо процес зіставлення завершився вчора, і ви скасовуєте його сьогодні, він повернеться вчорашнього успішного стану.

Система використовуватиме цю ліцензію для постійного збагачення ваших даних.

#### <a name="predictions"></a>Прогнози

- **Прогноз відтоку передплатників**

  Дотримуйтеся рекомендацій, щоб створити прогноз відтоку передплатників у таких секторах передплати, як хмарні служби, членство клієнта тощо. 

  Функція прогнозу відтоку передплатників дає змогу передбачити ймовірність того, що клієнт може припинити використання продуктів або послуг за підпискою, без залучення науковця в галузі даних. На основі прогнозної оцінки можна поєднувати іншу інформацію про клієнтів, щоб створити сегменти високого ризику відтоку. За допомогою цього сегмента можна звертатися до клієнтів безпосередньо у маркетингових сценаріях, для надання підтримки тощо, і зменшити ризик відтоку для конкретного клієнта та, відповідно, збільшити дохід і скоротити витрати.

  У межах досвіду можна уточнити визначення відтоку, як часове вікно, що стосується вашого бізнесу. Наприклад, для бізнесу з потокової передачі даних зі щомісячною передплатою відтоком клієнта може вважатися 15-денний період після закінчення терміну його підписки.

  Під час продовження роботи в прогнозом ми допоможемо вам у тому, які дані потрібні, а також допоможемо підставити дані щодо вашого бізнесу у поля, необхідні для прогнозування відтоку клієнтів. У разі зміни відомостей про комерційну діяльність бізнес-правила також можна вибрати розклад для повторного навчання з новою інформацією в системі для адаптації до нових обставин ведення бізнесу.    
  Додаткову інформацію див. у розділі [Прогноз відтоку передплатників (попередній перегляд)](predict-subscription-churn.md).

#### <a name="segments"></a>Сегменти

- **Пошук подібних клієнтів**
  
  Знайдіть схожих клієнтів у своїй базі клієнтів за допомогою штучного інтелекту. Модель машинного навчання бінарної класифікації визначає рівень подібності клієнтів у розширеному сегменті. Оцінка залежить від схожості на клієнтів з вихідного сегмента. Залежно від оцінки подібності профілі клієнтів додаються до новоствореного сегмента.

  Іноді, цей підхід називають "моделюванням за аналогією у цифровому маркетингу", і він передбачає застосування моделі ШІ для пошуку клієнтів, які подібні до інших сегментів ваших клієнтів, з урахуванням додаткових атрибутів. Він не тільки дозволяє вибрати атрибути, але також дає змогу вказати максимальну кількість клієнтів, які мають бути в цьому новому сегменті. Моделлю ШІ у подальшому обчислюється оцінка подібності для кожного клієнта відповідно до вибраних атрибутів і визначаються клієнти з вищою середньою оцінкою подібності. У сегмент з результатами буде включено клієнтів, які подібні на клієнта у вихідному сегменті.    
  Додаткові відомості див. у розділі [Подібні клієнти](find-similar-customer-segments.md).

- **Перекриття і відмінності сегментів**

  Перекриття сегментів дає змогу побачити, як багато і які клієнти є подібними до двох або більше сегментів. Наприклад, як сегмент споживачів високого рівня перекривається з сегментом клієнтів з високою вдоволеністю або як сегмент відтоку клієнтів перекривається з сегментом з низьким рівнем задоволеності. Крім того, можна проаналізувати як змінюється перекриття із застосуванням додаткового атрибута для вибору.

  Відмінності сегментів відображають відмінності одного з сегментів клієнтів від інших або від відмінності одного сегмента від іншого. Все, що потрібно зробити, це вказати сегмент. Система визначить атрибути та показники профілю, які відрізняють сегмент у формі рангового списку відмінностей: від найбільших відмінностей до найменших.    
  Додаткові відомості див. у розділі [Аналітичні огляди сегментів (попередній перегляд)](segment-insights.md).

- **Тривалість існування сегмента**
  
  Визначте розклад для активації або деактивації сегмента.    
  Додаткові відомості див. у розділі [Керування існуючими сегментами](segments.md#manage-existing-segments).

- **Автоматизація шляху до сутності**

  Під час створення сегмента користувачі мають визначити шлях сутності. Ця можливість – це перший крок у автоматизації визначення шляху до сутності, яка допомагає сфокусуватися на критеріях сегментації, визначених вами.    
  Якщо сутність, за якою потрібно сегментувати клієнтів, безпосередньо пов’язана з об’єднаною сутністю клієнта, більше не потрібно буде визначати шлях до сутності. Проте, якщо існує кілька можливих шляхів до сутності, їх потрібно визначити вручну.

- **Дії з кількома сегментами**
  
  Користувачі можуть вибрати кілька сегментів і виконати дії з ними (наприклад, оновити сегменти) одним рухом.    

- **Оновлення сегментів**

  Користувачі можуть оновити один сегмент або вибирати лише ті сегменти, які необхідно оновити.    

  
- **Вдосконалення складених сегментів**

  Користувачі можуть створювати, редагувати та видаляти сегменти, які базуються на інших сегментах. Наприклад, сегмент, побудований на іншому сегменті, який побудовано на третьому сегменті.    

- **Сторінка зі списком сегментів**

  У новому дизайні сторінки сегментів використовується формат списку, який дозволяє переглядати більше сегментів одночасно. Додано поле пошуку для швидкого пошуку сегментів. Тепер користувачі можуть застосовувати такі дії, як завантаження або видалення, одразу до кількох сегментів. Введено нову функцію тенденцій для швидкого визначення значних змін у сегментах.    
  Додаткові відомості див. в розділі [Створення сегментів і керування ними](segments.md).

- **Експорт до LiveRamp**

  Активуйте дані в LiveRamp®, щоб підключитися до більш ніж 500 платформ в різноманітних цифрових, соціальних і телевізійних екосистемах. Використовуйте дані в LiveRamp для націлювання, припинення та персоналізації рекламних кампаній.    
  Додаткову інформацію див. в розділі [З’єднувач LiveRamp&reg;](export-liveramp.md).

- **Надбудова Teams для Customer Insights**
  
  Бот дозволяє здійснювати пошук об’єднаних профілів клієнтів. Він відображає картку, яка містить до 15 полів з отриманого профілю клієнта. У разі декількох збігів відображається список результатів, у якому можна вибрати профіль.    
  Додаткові відомості див. в розділі [Бот Teams для Customer Insights](export-teams-bot.md).

#### <a name="measures"></a>Вимірювання

- **Сторінка зі списком показників**
  
  Удосконалення сторінки показників включають підтримку дій з одним показником та кількома показниками одночасно. Крім того, додано поле пошуку для швидкого пошуку та відстеження показників.    
  Додаткові відомості див. в розділі [Створення сегментів і керування ними](segments.md).

- **Вдосконалення складених показників**
  
  Користувачі можуть створювати, редагувати та видаляти показники, які базуються на інших показниках. Наприклад, показник, побудований на іншому показнику, який побудовано на третьому показнику.

#### <a name="segments"></a>Сегменти

- **Додатковий оператор**
  
  Оператор In-set дозволяє сегментацію для клієнтів за кількома можливими значеннями рядків. Перед тим, як було додано цей оператор, потрібно було створювати такі сегменти з кількома умовами АБО. Оператор In-set дозволяє робити це за допомогою однієї умови.    
  Додаткові відомості див. в розділі [Створення сегментів і керування ними](segments.md).

#### <a name="system-administration"></a>Адміністрування системи

- **Копіювання настройок конфігурації до нового середовища**
  
  Копіюйте конфігурацію з одного середовища до іншого. Під час створення нового середовища можна вибрати наявне середовище, з якого потрібно скопіювати конфігурацію. Наразі ми підтримуємо копіювання джерел даних, об’єднань даних, зв’язків, показників і сегментів. Облікові дані джерела даних та фактичні дані не копіюються.    
  Щоб отримати додаткові відомості, див. розділ [Керування середовищами](manage-environments.md).
