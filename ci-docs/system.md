# <a name="view-system-configuration"></a>Перегляд конфігурації системи

Перегляд системної інформації, стану системи та використання API.

## <a name="view-api-usage"></a>Переглянути використання API

Перегляньте детальну інформацію про використання API в режимі реального часу та подивіться, які події відбулися в певні часові рамки.

1. Перейдіть до **адміністративної** > **системи** та виберіть вкладку **використання** API.

1. **Виберіть часові рамки** для перегляду.

   Сторінка **використання** API містить три розділи:

   - **Виклики API** — діаграма, на якій графічно відображається об’єднана кількість викликів API в межах вказаних часових рамок.
   - **Передача даних** — діаграма, яка показує обсяг даних, які були передані через API в межах вибраних часових рамок.
   - **Операції** — таблиця, в якій рядки відповідають усім доступним операціям API, із докладними відомостям щодо цих операцій. Виберіть ім’я операції, щоб перейти до [довідника](https://developer.ci.ai.dynamics.com/api-details#api=CustomerInsights&operation=Get-all-instances) API.

   Операції, які використовують [прийом даних у реальному часі](real-time-data-ingestion.md), містять бінокулярний символ для перегляду використання API в режимі реального часу.

   1. Виберіть бінокль, щоб відкрити панель використання **API в режимі реального** часу, що містить відомості про використання операції.
   1. **Виберіть часові рамки** для перегляду.
   1. Скористайтеся **полем Група за**, щоб вибрати, як найкраще презентувати взаємодію в режимі реального часу. Згрупуйте дані за методами API, кваліфікованим ім’ям **сутності (проковтнутою сутністю),** створеним (джерелом події), **результатом** (успіхом або невдачею) або **кодами** **помилок.** **·** Дані доступні у вигляді діаграми та таблиці.

## <a name="view-system-information"></a>Перегляд відомостей про систему

Перегляньте коротке ім’я середовища, ідентифікатор, регіон, тип та ідентифікатор сеансу.

1. Перейдіть до **системи адміністрування** > **та** виберіть вкладку **Про програму**.

1. Щоб переглянути мову та країну або регіон, перейдіть на вкладку **Загальні**.

### <a name="update-preferred-language-or-countryregion"></a>Оновлення бажаною мовою або країною або регіоном

Customer Insights [підтримує багато мов](/dynamics365/get-started/availability). Програма використовує мовні параметри для відображення таких елементів, як меню, текст підписів і повідомлення, вашою основною мовою.

Імпортовані дані та введені вручну дані не перекладаються.

1. Перейдіть в **Admin** > **System** і виберіть вкладку **Загальні**.

1. Щоб змінити основну мову, виберіть **Мова** в розкривному списку.

1. Щоб змінити бажане форматування дат, часу й чисел, скористайтеся розкривним списком **Формат країни/регіону**. Відобразиться попередній перегляд форматування. Система автоматично пропонує вибір при виборі нової мови.

1. Виберіть **Зберегти**.

## <a name="view-system-status"></a>Переглянути стан системи

Відстежуйте хід виконання завдань, прийом даних, експорт даних і кілька інших важливих процесів продукту. Перегляньте інформацію, щоб переконатися в повноті ваших активних завдань і процесів.

1. Перейдіть до **адміністративної** > **системи** та виберіть вкладку **Стан**.

   Відображення стану та обробки інформації для різних процесів. Перегляньте ім’я **завдання**, **стан** останнього запуску та дату останнього **оновлення**.

1. Щоб переглянути відомості про кілька останніх запусків, виберіть завдання або ім’я процесу.

1. Щоб переглянути відомості про перебіг виконання завдання, виберіть стан. Відобразиться **область Відомості про** перебіг виконання.

   :::image type="content" source="media/system-progress-details.png" alt-text="Область відомостей про перебіг виконання системи":::

1. Щоб переглянути відомості про перебіг виконання для всіх завдань, виберіть пункт **Увесь робочий цикл**.

### <a name="status-definitions"></a>визначення статусу

Система використовує наступні статуси для завдань і процесів:

|Статус  |Визначення  |
|---------|---------|
|Скасовані |Завдання або процес були скасовані користувачем до його завершення.   |
|Не пройдено   |Завдання або процес зіткнулися з помилками.         |
|Помилка  |Завдання або процес не вдалося виконати.  |
|Не розпочато   |джерело даних ще немає даних, або завдання все ще знаходиться в режимі чернетки.         |
|Обробна промисловість  |Виконується завдання або процес.  |
|Оновлення    |Виконується завдання або процес. Щоб скасувати цю операцію, виберіть елемент **Оновити** та **Скасувати завдання**. Зупинивши оновлення завдання або процесу, ви повернете його до стану останнього оновлення.       |
|Пропущено  |Завдання або процес було пропущено. Один або кілька процесів, від яких залежить це завдання, завершилися помилкою або були пропущені.|
|Успішно  |Завдання або процес успішно виконано. Для джерел даних указує, що дані успішно відібрали, якщо в стовпці **Оновлено** вказано час.|
|Поставлено в чергу | Обробка стоїть в черзі і почнеться після завершення всіх завдань і процесів. Щоб отримати додаткові відомості, перегляньте статтю [Оновлення процесів](#refresh-processes).|

### <a name="refresh-processes"></a>Оновлення процесів

Оновлення для завдань і процесів виконується за налаштованим [розкладом](schedule-refresh.md).

|Обробити  |Опис  |
|---------|---------|
|Активність  |Виконується вручну (одноразове оновлення). Залежить від процесу об’єднання. Insights залежать від виконання.|
|Зв’язування аналізу |Виконується вручну (одноразове оновлення). Залежить від сегментів.  |
|Підготовка аналізу |Виконується вручну (одноразове оновлення). Залежить від сегментів.  |
|Підготовка даних   |Потрібна сутність, щоб працювати далі. джерело даних сутності залежать від прийому всередину. Збагачені суб’єкти залежать від збагачення. Сутність клієнта залежить від об’єднання.  |
|Джерела даних   |Не залежить від будь-якого іншого процесу. Відповідність залежить від успішного завершення цього процесу.  |
|Збагачення   |Виконується вручну (одноразове оновлення). Залежить від процесу об’єднання. |
|Напрямки експорту |Виконується вручну (одноразове оновлення). Залежить від сегментів.  |
|Аналітика |Виконується вручну (одноразове оновлення). Залежить від сегментів.  |
|Аналітика   |Залежить від злиття.   |
|Збіг |Залежить від обробки джерел даних, що використовуються в описі відповідності.      |
|Вимірювання  |Виконується вручну (одноразове оновлення). Залежить від процесу об’єднання.  |
|Злиття   |Залежить від завершення процесу відповідності. Сегменти, показники, збагачення, пошук, справи, прогнози та підготовка даних залежать від успішного завершення цього процесу.   |
|Профілі   |Виконується вручну (одноразове оновлення). Залежить від процесу об’єднання. |
|Введіть пошуковий запит   |Виконується вручну (одноразове оновлення). Залежить від процесу об’єднання. |
|Сегменти  |Виконується вручну (одноразове оновлення). Залежить від процесу об’єднання. Insights залежать від виконання.|
|Система   |Залежить від завершення процесу відповідності. Сегменти, показники, збагачення, пошук, справи, прогнози та підготовка даних залежать від успішного завершення цього процесу.   |
|User  |Виконується вручну (одноразове оновлення). Залежить від сутностей.  |

Виберіть стан процесу, щоб переглянути відомості про перебіг виконання всієї роботи, у якій він брав участь. Наведені вище процеси оновлення можуть допомогти зрозуміти, що можна зробити для вирішення пропущеного **завдання** або **процесу в черзі**.
